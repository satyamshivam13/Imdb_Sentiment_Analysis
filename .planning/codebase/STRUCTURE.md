## Directory Purposes

**Root (`.`):**
- Purpose: Main runnable web app and deployment artifacts.
- Contains: Flask server module, templates, static assets, ML artifacts.
- Key files: `app.py`, `requirements.txt`, `Procfile`.

**`templates/`:**
- Purpose: Server-rendered HTML views.
- Contains: base layout and page templates.
- Key files: `templates/base.html`, `templates/home.html`, `templates/result.html`.

**`static/`:**
- Purpose: Frontend resources delivered directly by Flask.
- Contains: CSS and JavaScript.
- Key files: `static/css/app.css`, `static/js/app.js`.

**`Imdb-Sentiment-Analysis-Flask/` (nested):**
- Purpose: Secondary copy of the same project.
- Contains: another app entrypoint, duplicate assets, and a full `venv/`.
- Key files: `Imdb-Sentiment-Analysis-Flask/app.py`, nested `requirements.txt`.

## Naming Conventions

**Files:**
- Python modules use snake_case names (`app.py` as single module).
- Templates use lowercase names by page purpose (`home.html`, `result.html`).
- Static asset names are short and generic (`app.css`, `app.js`).

**Directories:**
- Conventional Flask layout (`templates/`, `static/`).
- Duplicated top-level project folder introduces ambiguous ownership.

## Where to Add New Code

**Server features:**
- Add to root `app.py` for now (until server code is modularized).

**UI updates:**
- Templates in root `templates/`.
- Styles in root `static/css/app.css`.
- Interactions in root `static/js/app.js`.

**Tests (new):**
- Recommended new location: `tests/` at repo root.
